jdbc overview

jdbc allows java applications to connect to a relational database(oracle,sql server etc).

java application -> jdbc -> db(local,network,cloud)

Features include:

provides portable access to various databases - no need to develop code for different databases.

jdbc is also a call level interface to teh db - it supports ansi sql 2003, you simply provide sql statements to teh jdbc driver and it will pass those to the actual db.can build custom sql statements like insert,select,update,delete, complex sql queries inner/outer joins,can call stored procedures.

jdbc architecture

java application -> jdbc driver -> db

jdbc arch includes a key component, ie jdbc driver,it provides the connection to the db.also the jdbc driver converts the standard api calls to low level calls for your specific db.

the database vendor normally provides the jdbc driver file(jdbc driver implementation) for you.

jdbc driver manager

dm helps to connect an application to db based on the database connection string.

in jdbc 4 the jdbc drivers are automatically loaded based on the classpath.With jdbc 3 the drivers have to be explicitly loaded with Class.forName(theDrivername).

jdbc api is defined in the following packages. - java.sql,javax.sql

key classes: java.sql.DriverManage,java.sql.Connection,java.sql.Statement,java.sql.ResultSet,javax.sql.DataSource(for connection pooling)
